Reference documentation for the tools and infrastructure that make up the Clef ecosystem.

Clef is designed to be compiled, linked, and distributed without runtime dependencies. That goal shapes every layer of the toolchain — from how packages are resolved at the source level, to how assemblies are structured for native deployment, to how ClefPak orchestrates builds across heterogeneous targets. The articles here cover these mechanisms in detail.
